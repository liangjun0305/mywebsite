2.2. Options

图 14.7. Nessus策略配置之Options


Options所有的选项都描述在下面的表格中：

表 14.1. Options所有的选项

选项名称选项描述
Number of hosts in parallel设置最多有多少主机能被同时扫描
Number of checks in parallel设置最多同时运行的插件数量，Nessus扫描的速度是非常快的，但是由于网络的限制，特别是广域网，你需要减少同时运行插件的数目，来优化Nessus的运行。
Port scanner range设置端口扫描的范围，当你需要扫描特定的端口，这个选项就非常有用，默认端口扫描的范围定义在nessus-services文件中。这个选项的格式可以是范围如：137-139,也可以是单个加范围137-149,80。
Safe checks当你扫描时有些会引起被扫描主机出现一些特殊症状，Nessus会识别出这些主机，你只需要选上这个选项，Nessus就不会扫描这些主机。例如，扫描一台打印服务器，可能会引起打印服务器的重启。利用Safe Checks这个选项阻止扫描那些被探测为打印服务器的主机。
Designate hosts by their DNS nameEnables the ability to specify a list of DNS named assets as your “Network(s) to scan” on the Scan tab rather than a single IP address or IP address ranges.(这是英文原文，暂时没有理解，希望大家帮助)。
Consider unscanned ports as closed当你指定端口进行扫描时候，如果你不需要扫描其他端口的插件进行运行，你可以使用这个选项，节省资源。
Save knowledge base on disk些个选项告诉扫描者将扫描的信息保存到扫描服务器中以便日后使用。
Save a packet capture of the scan保存你扫描其间的TCP流量，这个记录可以帮助你来分析扫描其间的网络问题。保存的文件你可以用Tenable的其他产品来分析，具体见其官方网站。
Log details of the scan on the server在Nessus服务器保存你扫描的细节，这些记录你扫描用到的插件和扫描目标。
Port scanners to use：这部分的选项允许你选择扫描的方式。
Nessus SNMP scanner这个选项将会向扫描目标寻求SNMP的回复，扫描过程中Nessus会根据Advanced表中的SNMP setting来进行探测。
Nessus TCP Scanner利用TCP扫描来扫描目标的开放端口。
Netstat ‘scanner’这个选项是利用Netstat来探测目标开放的端口，这个主要基于UNIX系统。
Ping the remote host这个选项是在扫描端口之前对目标主机执行ping，来探测目标主机是否存在。
SYN Scan一种扫描端口的方式，向目标端口发送一个SYN包，等待目标端口返回ACK包，如果目标端口在一定的等待时间内没有返回，则判断这个端口是关闭的。这个方式在对通过一个防火墙的扫描是非常实用的。
Scan for LaBrea tarpitted hostsLaBrea tarpits是蜜罐的一种形式，他的部署是为了减少扫描和给扫描一些错误的信息，你选上此选项，Nessus将尝试去识别这些LaBrea tarpits系统并放弃对他们的扫描。

2.3. Credentials

图 14.8. Credentials选项


Credentials选项有三个子选项：Windows credentials、SSH settings、Kerberos configuration。

Credentials选项需要你提供你扫描的服务器的SMB和SSH的用户和口令，来检测你SMB和SSH是否有重要的安全漏洞。

2.4. Plugin Selection

Plugin Selection选项支持用户进行个性化的安全检查，你可以通过选择大类和小类的插件形式来完成你的个性化。

图 14.9. Plugin Selection选项


Plugin Selection选项首先显示的是所以大类插件，你可以通过+来展开查看里面的小类插件。

如果大类插件勾选上，里面小类也都全部勾选上，那么这个大类来新插件后，也将被默认勾选上。如果大类没有勾选上，那么这个大类来新插件后，也将默认不被勾选上。如果大类没有勾选上，里面小类有些被勾选上了，而有些没有被勾选上，那么这个大类来新插件后，也将默认不被勾选上。

注意
“Denial of Service”这个大类在你Options->Safe Checks没有打开，会引起一些企业网络的异常。但是这里面也有一些不会引起危害的有用检查。建议“Denial of Service”不要用于生产网络。

2.5. Network

Network选项能让你以最小的网络代价产生最好的扫描结果，默认Nessus将会使用你所有的硬件资源和占用你的所以网络，这样会造成一些机器的超负荷和一些响应超时，这些选项将会很好的微调Nessus来达到最好的效率。

图 14.10. Network选项


表 14.3. Options所有的选项

选项名称选项描述
Reduce the number of connections in parallel on congestionNessus扫描时候会在极短的时间发送大量的数据包，你打开了这个选项，如果你但Nessus的大量包把网络阻塞，那么Nessus将会调整发包来减轻网络负载，当Nessus发现网络顺畅后，那么Nessus又将自动调整，适应网络。
Use the kernel congestion detection (Linux only)这个选项只适用与Linux系统，这个选项将会使Nessus检测CPU等内部资源，做到合理利用。
Stop scanning hosts turned off during the audit此选项用来检测如果你扫描中，目标主机忽然关机或是其他原因没有响应，而停止扫描目标主机。
Max simultaneous TCP sessions per scan此选项是限制整个扫描过程中同时发出的TCP数据包的数量，设置一个比较小的数量有可能使你的扫描结果更加真实，特别是在一些限速的网络中。
Max simultaneous TCP sessions per host这个选项跟上面一样，不过是限制对每一个主机发送数据包的限制。
Network receive timeout (in seconds)默认设置是5秒，此选项是设置Nessus等待目标主机回应的时间，如果某个插件对回复时间有特别的指出除外。如果你在一个很慢的网络中扫描，你可以把整个设置写的时间大点
