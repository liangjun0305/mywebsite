1. 创建用户

由于Nessus采用的服务器/客户端的方式，所以你必须在服务器这边创建用户让客户端来登录上来进行扫描和检索扫描结果。

注意
下面的每一个命令都是使用的全路径来执行的，你不可以用这种方式：先进入脚本目录，然后使用./老执行命令。
注意
除非特别指出，不然下面的所有命令都是在root权限下运行。

密码验证方式使用nessus-adduser来添加用户和密码，第一个被创建的应该是admin用户。

每一个Nessus用户都可以通过“user rules”的规则来控制他们能扫描什么或是不能扫描什么。默认的规则是用户可以进行任意的扫描。

注意
下面命令都是假定Nessus安装目录是/opt/nessus，如果你是FreeBSd，将命令行中的/opt/nessus替换成/usr/local/nessus/就行了。
# /opt/nessus/sbin/nessus-adduser
Using /var/tmp as a temporary file holder
Add a new nessusd user
--------------------
Login : admin
Authentication (pass/cert) [pass]:
Login password:
Login password (again):
User rules
----------
nessusd has a rules system which allows you to restrict the hosts
that admin has the right to test. For instance, you may want
him to be able to scan his own host only.
Please see the nessus-adduser(8) man page for the rules syntax
Enter the rules for this user, and hit ctrl-D once you are done:
(the user can have an empty rules set)
Login : admin
Password : ***********
DN :
Rules :
Is that ok ? (y/n) [y]
user added.

一个Nessus服务器端能够支持多用户一些复杂安排，比如，可能一个组织需要多个用户来访问一个Nessus服务器，但是这些用户能够扫描不同的IP地址范围，下面的例子就是使用密码验证的方式创建第二个用户，用户规则（限制用户扫描C类网段192.168.2.0/24）。 更多的说明请看nessus-adduser的man手册。

# /opt/nessus/sbin/nessus-adduser
Using /var/tmp as a temporary file holder
Add a new nessusd user
--------------------
Login : restricteduser
Authentication (pass/cert) [pass]:
Login password:
Login password (again):
User rules
----------
nessusd has a rules system which allows you to restrict the hosts
that restricteduser has the right to test. For instance, you may want
him to be able to scan his own host only.
Please see the nessus-adduser(8) man page for the rules syntax
Enter the rules for this user, and hit ctrl-D once you are done:
(the user can have an empty rules set)
accept 192.168.2.0/24
default deny
Login : restricteduser
Password : ***********
DN :
Rules :
accept 192.168.2.0/24
default deny
Is that ok ? (y/n) [y]
User added.
注意

在一些操作系统中看nessus-adduser(8)的man手册，需要执行如下的命令：

export MANPATH=/opt/nessus/man

man nessus-adduser
