2. 从Nessus 2.x 升级到Nessus 3.x

这一节将讲述从Nessus 2.x 升级到Nessus 3.x的过程。

Nessus 3.x的安装目录（FreeBSD安装在/usr/local/nessus/，其他的都安装在/opt/nessus/）和Nessus 2.x不同，你必须手动把一些需要的文件从Nessus 2.x的目录拷到Nessus 3.x的目录。

注意
下面的一些示例都假定Nessus 2安装在/user/local/下面

现在最新版的Nessus http://www.nessus.org/download/。

2.1. FreeBSD 平台

Nessus 2.x 和Nessus 3.x是安装在不同的路径，所以它们能够同时安装在同一个系统里。从Nessus 2.x 升级到Nessus 3.x的第一步就是停止你现在Nessus 2.x的服务，用如下命令：

# killall nessusd
注意
killall 这个命令会停止正在进行的扫描

下一步安装Nessus 3.x，安装步骤请查阅安装部分。

在下一步，将Nessus 2.x的用户转移到Nessus 3.x上，用户管理目录在Nessus目录下。所以转移将是非常简单，用如下命令：

# cp –r /usr/local/var/nessus/users/* /usr/local/nessus/var/nessus/users/

然后，你需要把nessus-fetch.rc拷到Nessus 3.x的相应目录下，这个文件保存的是插件激活码，如果你不想要这么做，那么你只有去Nessus网站重新注册激活码。

# cp /usr/local/etc/nessus/nessus-fetch.rc /usr/local/nessus/etc/nessus/

确认下权限是不是如下：

-rw------- 1 root root 398 Nov 3 03:12 nessus-fetch.rc

下一步编辑/usr/local/nessus/etc/nessus/nessusd.conf，确定如下选项是否填写正确：

plugin_upload = yesadmin_user = <ADMIN>

<ADMIN>就是管理员用户，在Nessus 2.x中定义在/usr/local/etc/nessus/nessud.conf。

现在你可以配置Nessus 3.x了，首先启动Nessus程序，然后通过Nessus的客户端进行配置，这个你可以查看。

最后，当你配置完成Nessus 3.x，而且运行正常后，你可以将Nessus 2.x卸载：

# /usr/local/sbin/uninstall-nessus
