1. 测试高可用

做到高可用实际就是当一台节点宕机之后，整个集群仍然能提供服务。

我们这里测试过程是，rac-master正在提供服务，出现突然宕机，然后rac-slave将rac-master的服务接管过来。

首先配置客户端的tnsnames.ora文件

demo =
  (DESCRIPTION =
    (ADDRESS = (PROTOCOL = TCP)(HOST = 192.168.1.63)(PORT = 1521))
    (ADDRESS = (PROTOCOL = TCP)(HOST = 192.168.1.64)(PORT = 1521)) 
    (LOAD_BALANCE = yes)  
    (CONNECT_DATA =
      (SERVER = DEDICATED)
      (SERVICE_NAME = demo)
    )
  )

然后连接到rac-master上

sqlplus system/password@demo

查看当前实例：

SQL> select instance_name from v$instance;
INSTANCE_NAME
--------------------------------
demo1

然后将rac-master关机，再当前实例：

SQL> select instance_name from v$instance;
INSTANCE_NAME
--------------------------------
demo2
