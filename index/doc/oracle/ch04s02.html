在rac-slave运行root.sh最后会有error，没关系，这些vipca的bug，我们来手动运行vipca。

在执行vipca之前，需要执行几行命令帮助vipca来识别两个网卡的作用。

root@rac-slave:~# cd /var/oracle/product/crs/bin/

root@rac-slave:/var/oracle/product/crs/bin# ./oifcfg iflist

eth3  192.168.2.0
eth2  192.168.1.0

root@rac-slave:~# ./oifcfg setif -global eth2/192.168.1.0:public

root@rac-slave:~# ./oifcfg setif -global eth3/192.168.2.0:cluster_interconnect

注意
如果你两台服务器的网卡的名字都一样，都是eth0和eth1，或是都是eth2和eth3，那么只需在rac-slave中运行以上命令即可，但是如果两台服务器的网卡名字不一样，我这个环境就是不一样，那么你需要在rac-master中运行下面的命令：

root@rac-master:~# cd /var/oracle/product/crs/bin/

root@rac-master:/var/oracle/product/crs/bin# ./oifcfg iflist

eth0  192.168.1.0
eth1  192.168.2.0

root@rac-master:/var/oracle/product/crs/bin# ./oifcfg setif -global eth0/192.168.1.0:public

root@rac-master:/var/oracle/product/crs/bin# ./oifcfg setif -global eth1/192.168.2.0:cluster_interconnect

然后在rac-slave的图形界面中运行一个root终端：

root@rac-slave:~# /var/oracle/product/crs/bin/vipca
